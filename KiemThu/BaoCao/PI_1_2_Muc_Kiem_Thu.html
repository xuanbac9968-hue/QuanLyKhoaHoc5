📊 Bốn Mức Kiểm Thử – Tổng Quan

🔬 Unit Testing

  • Mục tiêu: Kiểm tra từng class/method riêng lẻ
  • Công cụ: xUnit 2.9, Moq 4.20, coverlet 6.0
  • Số lượng: 531 tests (0 failed)
  • Coverage: 82.9% line coverage
  • Chạy bởi: Developer (CI tự động)
  • Tần suất: Mỗi commit/push

🔗 Integration Testing

  • Mục tiêu: Kiểm tra tương tác giữa các layer
  • Phạm vi: Controller↔Service↔Repository↔DB
  • Công cụ: xUnit + In-Memory DB, Postman
  • Mô-đun test: Chat API, ThanhToan flow, DangKy pipeline
  • Chạy bởi: Developer + CI
  • Tần suất: Mỗi sprint

🌐 System Testing

  • Mục tiêu: Kiểm tra toàn bộ hệ thống end-to-end
  • Công cụ: JMeter (load), ZAP (security), SonarQube (quality)
  • Load test: 100/200/300 concurrent users
  • Security: 0 High vulnerabilities
  • Chạy bởi: QA team
  • Tần suất: Trước release

✅ Acceptance Testing (E2E)

  • Mục tiêu: Xác nhận hệ thống đáp ứng yêu cầu business
  • Công cụ: Katalon Studio 11.1.3.0, Chrome 148
  • Test Suite: TS_All (6 test cases)
  • Kết quả: 6/6 PASSED (377s)
  • Chạy bởi: QA + Product Owner
  • Tần suất: Sprint review, Release
